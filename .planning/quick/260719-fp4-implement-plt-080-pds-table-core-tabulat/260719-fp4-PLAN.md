<objective>
Implement PLT-080 (issue #7): the Pealton Design System `pds-table` core — a
framework-free, zero-runtime-dep Tabulator wrapper `createPdsTable(el, options)`
that applies the DS theme, is RTL-safe with virtual scroll and no pagination,
renders the four contract states via an overlay, and exposes a selection +
bulk-bar API. Add supporting CSS to the existing `tabulator.css`, a live demo
`docs/table.html`, and a source-level test file.

Purpose: give both Pealton apps ONE themed table component so Epic 3 adoption
tasks (PLT-084..088) wire behaviour instead of rebuilding it per surface.
Output: `pds-table.js`, appended `tabulator.css`, `docs/table.html`,
`tests/test_table_module.py`.

Scope guards (do NOT build): inline-edit cells (PLT-081), data fetching /
progressive-load wiring (PLT-083), replacing Tabulator, real-estate tables. The
wrapper is app-agnostic — NO candidate fields, NO HR endpoints, NO Hebrew
business strings baked in. The ONLY allowed built-in Hebrew is the selection-bar
"N נבחרו" count label (contract Selection section).
</objective>

<task type="auto">
  <name>Task 1: Core wrapper — createPdsTable, escape helper, theme + Tabulator init, selection</name>
  <files>pealton_design_system/static/pds-table.js</files>
  <action>
Create the new file as a single vanilla-JS IIFE opening with "use strict",
matching the header-comment and style of pds.js (PLT-059). ZERO runtime deps;
assumes global `Tabulator` is already loaded by the consuming app.

Define an internal `escapeHtml(value)` helper: coerce to string, replace the
five characters `&`, `<`, `>`, `"`, `'` with their HTML entities. This is the
single sanitiser every built-in formatter and every injected state string MUST
route through (contract Rendering-safety cross-cutting rule; project security
posture — escape all text).

Define `createPdsTable(el, options)`:
  - Resolve `el` (accept a DOM node or a selector string).
  - Add the class `pds-tabulator` to the element so the DS theme applies (theme
    selectors are `.pds-tabulator.tabulator ...`).
  - Merge caller options over DS defaults, DS defaults being overridable:
    `textDirection` rtl (RTL-safe default), `renderVertical` virtual (virtual
    scrolling ON), `layout` fitColumns. Do NOT set any pagination option and
    force pagination off (progressive-scroll model per contract "Progressive
    loading" — NO pagination). Pass caller `columns`/`data` through.
  - Selection: default `selectableRows` true (overridable). Read a
    caller-configurable id field via `options.idField` (default the string id).
  - Prepend a selection column when selection is enabled, using
    `formatter` rowSelection, `titleFormatter` rowSelection, `headerSort` false,
    centered, fixed narrow width.
  - Selection payload + bulk-bar mount: on Tabulator `rowSelectionChanged`,
    compute `selectedIds` (map selected rows' data through `idField`) and
    `count`. Expose `getSelectedIds()`, `clearSelection()` (Tabulator
    `deselectRow()`), and fire `options.onSelectionChange({ selectedIds, count,
    clear })` where `clear` is `clearSelection`. If a bulk-bar mount element is
    provided (`options.bulkBar` node or selector) or auto-created, keep a count
    label element and set its text to the count followed by the built-in Hebrew
    word — i.e. render "N נבחרו" — and toggle the bar's visibility on
    count>0. Consumers render their OWN action buttons into the mount; the DS
    provides only the mount + count label + a clear affordance (contract
    Selection — DS provides mount + count + clear only).
  - Return a controller object exposing at least: `table` (the Tabulator
    instance), `getSelectedIds`, `clearSelection`, `bulkBar` (the mount element),
    and `destroy`. Leave a clear seam so Task 2 attaches setState + state methods
    to this same controller.
  - Expose the API globally BOTH ways: `window.createPdsTable = createPdsTable`
    and, mirroring pds.js, set `window.PDS = window.PDS || {}` then
    `window.PDS.createTable = createPdsTable`.

Document each block with a short purpose comment (clean/modular/documented, no
clever tricks — CLAUDE.md). Do NOT bake in any candidate/HR field names or any
external URL.
  </action>

<task type="auto">
  <name>Task 2: State controller — overlay with skeleton / empty / error+retry / loading-more</name>
  <files>pealton_design_system/static/pds-table.js</files>
  <action>
Extend the same file (do not create a second file). Inside `createPdsTable`,
after the table element is resolved, create ONE overlay element (a div with
class `pds-table-overlay`, initially hidden) positioned over the table container
so it composes on top of the themed Tabulator without altering table markup.

Add a `setState(name)` method to the controller accepting exactly these five
names: loading, empty, error, loading-more, ready. Behaviour per the contract
States section:
  - ready: hide the overlay entirely (rows show through).
  - loading: show a skeleton — a small set of placeholder skeleton rows
    (classes such as `pds-table-skeleton` / `pds-table-skeleton__row`) in place
    of rows (first-load skeleton).
  - empty: show per-view empty copy supplied by the caller (`options.emptyText`,
    default a neutral non-business fallback — NOT a Hebrew business string). The
    copy MUST be inserted via `escapeHtml` (Task 1 helper). Empty message is
    per-view, not a baked generic (contract: per-view copy).
  - error: show error copy (`options.errorText`, escaped) plus a retry
    affordance (a button) that invokes the caller's retry callback. Support both
    an `options.onRetry` default and a `showError(text, onRetry)` convenience
    that overrides per call. Retry button click calls the provided callback
    (contract: error state exposes a retry affordance).
  - loading-more: show a partial / in-flight indicator (a small spinner, class
    such as `pds-table-spinner`) layered NEAR the loaded edge WITHOUT hiding
    already-loaded rows, so the table does not look finished (contract:
    progressive-load partial indicator).

Add thin convenience methods on the controller that delegate to setState:
`showLoading()`, `showEmpty(text)`, `showError(text, onRetry)`,
`showLoadingMore()`, `showReady()`. All caller-supplied text (empty copy, error
copy) routes through `escapeHtml` before insertion — never string-concatenate
raw caller text into innerHTML.

Keep the state DOM minimal and token-styled via the CSS added in Task 3; JS sets
only classes/text, not inline colours.
