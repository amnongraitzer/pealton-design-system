<objective>
Adopt Web Awesome Free (MIT, v3.10.0) into the Pealton Design System, **vendored**
(no CDN, no autoloader), themed onto the existing `--pds-*` tokens, and RTL-ready —
mirroring the self-hosted Tabulator / pds-table.js pattern already in the repo.

Purpose: give both Pealton apps a shared, DS-themed set of richer interactive
components (searchable multi-select, dialog, input, button) without a framework or
a consumer build step. Web components self-register on import and read design
decisions from cascading `--wa-*` custom properties, so a thin token-remap layer
makes them look native to the DS.

Output: three vendored static artifacts + a hand-written DS theme layer, a proving
demo page, a consumer doc, and a source-level pytest.

**No build step for consumers.** They load static files exactly as they load
`tokens.css` today. The esbuild step in Task 1 is a ONE-TIME vendoring step that
produces static artifacts — the same way Tabulator's `tabulator.min.js` was produced
upstream. It is never run by a consuming app.
</objective>
